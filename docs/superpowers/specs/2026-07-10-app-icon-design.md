# Sotto App Icon — Design

**Date:** 2026-07-10
**Status:** Approved (brainstormed via visual companion; mark, palette, and format selected by Connor)

## Goal

Give Sotto a real app icon. The app currently has no asset catalog and no icon of any
kind — it ships with the default blank grid. One icon design must cover the main app,
the widget extension (which displays the parent app's icon), and the App Store 1024 px
marketing icon, and must render correctly in all six iOS 26 appearance modes: Default,
Dark, Clear (light/dark), and Tinted (light/dark).

## Concept: "Wave becomes text"

The icon is a paragraph. The first line begins as a decaying speech waveform and
settles into a ruled line; below it sit a full transcript line and a shorter closing
line (a paragraph's last line). It tells Sotto's story literally — speech in, markdown
out — and was chosen over "submerged wave" and hybrid concepts after small-size
legibility comparison.

Differentiation rationale: every competing voice app (including the other apps named
Sotto) uses a microphone, speech bubble, or loud full-height waveform. This mark avoids
all three; it is mostly quiet lines, matching an app that records only when people talk.

## Geometry

Canvas: 160-unit square reference space (scale ×6.4 for a 1024 px canvas). All lines:
width 8, round caps, single color.

| Element | Centerline (160-unit space) | Opacity |
|---|---|---|
| Speech line | `M26 52 C28.5 47.5 32 42 36 42 C44 42 46 60 54 60 C61 60 61 47 68 47 C72 47 74 52 78 52 L134 52` | 1.0 |
| Transcript line | `(26,86) → (134,86)` | 0.8 |
| Closing line | `(26,120) → (98,120)` | 0.8 |

Content margins: 26 units (~16 %) on each side. The 80 % opacity on the two text lines
keeps the spoken line dominant; opacity survives Tinted mode because the system maps
glyph luminance to tint intensity.

Two constraints discovered during implementation (2026-07-10) amended the original
four-wiggle wave:

1. **Curvature floor.** A stroked curve renders cleanly only while its radius of
   curvature stays above the stroke half-width (4 units). The original wave's tight
   oscillations (R ≈ 0.5) made the stroke fold into itself and fused the humps into
   one slab under Liquid Glass. The wave above keeps R ≥ ~4.8 everywhere: one strong
   oscillation and a gentle echo, decaying amplitudes 10/8/5, landing tangent-horizontal
   into the rule.
2. **Filled outlines, not strokes.** Icon Composer's Liquid Glass renderer breaks on
   stroked SVG paths (hollow centers, gaps); layer SVGs must be filled shapes. The
   glyph SVG is therefore the *outline* of the centerlines above (offset ±4, round
   caps), generated by `tools/outline_glyph.py` — rerun it after any geometry change.
   Line opacity is carried as `fill-opacity`.

## Palette: "Porcelain"

Chosen over warm cream (rejected as an overused default and too yellow) and over flat
pure white and cool frost variants.

| Mode | Background | Glyph |
|---|---|---|
| Default (light) | Vertical gradient `#FDFDFC → #F2F2EF` | `#232752` indigo ink |
| Dark | Vertical gradient `#1A1D3C → #0B0C20` | `#F2F2EF` |
| Clear light/dark | System glass (background discarded) | System-rendered from glyph |
| Tinted light/dark | System glass | User accent, from glyph luminance |

The identity is white + indigo ink — a miniature of the app's stark white transcript
interface. Paper by day, ink by night.

## Deliverable format

An **Icon Composer `.icon` bundle** at `Sotto/AppIcon.icon` — a directory containing
`icon.json` plus SVG layer assets:

- Layer 1 (background): the Porcelain gradient, with the dark-mode gradient as its
  dark-appearance override.
- Layer 2 (glyph): the three lines as a single mono-color SVG of filled outline paths,
  with light/dark fill overrides. Being a single-color glyph is what guarantees Clear
  and Tinted modes work without additional artwork. The layer's group carries Icon
  Composer's standard material defaults (neutral shadow 0.5, specular, translucency
  0.4, individual lighting), matching how Icon Composer authors new documents.

Schema note: when a fill varies by appearance, *all* appearances live in
`fill-specializations` (the light entry simply omits the `appearance` key) and the
plain `fill` key is dropped — Icon Composer ignores specializations if a base `fill`
is also present.

Tinted note: Tinted mode maps glyph luminance to tint intensity, so the indigo ink
(luminance ≈ 0.15) rendered nearly invisible. The glyph layer carries an explicit
`tinted` specialization of full-brightness white (`extended-gray:1,1`); the text
lines' 0.8 alpha still provides the hierarchy.

The bundle is authored programmatically (JSON + SVG are plain text, git-friendly) and
can be opened in Apple's Icon Composer app (ships with Xcode 26) to preview true
Liquid Glass rendering or hand-tune material properties later. Both workflows edit the
same file.

**Fallback:** if the programmatic `icon.json` fights us on undocumented format details,
assemble the bundle manually in Icon Composer from the same SVG layers (~5 minutes).
The design is unchanged either way.

## Build integration

The project is generated by XcodeGen, so integration happens in `project.yml`:

1. Add `Sotto/AppIcon.icon` to the Sotto target (resource; excluded from the plain
   source glob if needed, mirroring how the CoreML model folder is handled).
2. Set `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` in the target's settings.
3. `xcodegen generate`.

No asset catalog is required. The widget extension needs no icon of its own; the
App Store icon is derived from the same bundle at archive time.

## Verification

1. Build to simulator or device; confirm the icon appears (no blank grid).
2. Long-press wallpaper → Customize: check all four home-screen appearance settings
   (Default, Dark, Clear, Tinted).
3. Check Settings and Spotlight for small-size legibility.
4. Open `AppIcon.icon` in Icon Composer and confirm it parses and previews cleanly.

## Out of scope

- In-app iconography, launch screen, marketing/web assets.
- Widget-specific icons (WidgetKit uses the app icon).
- Restoring the iCloud entitlements (unrelated; tracked in issue #5).
