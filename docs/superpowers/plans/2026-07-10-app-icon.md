# Sotto App Icon Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Sotto's "wave becomes text" app icon as an Icon Composer `.icon` bundle, wired into the XcodeGen build so it renders in all six iOS 26 appearance modes.

**Architecture:** A two-layer Icon Composer document — a top-level Porcelain gradient background with a dark-appearance specialization, plus a single mono-color SVG glyph layer (three strokes) with light/dark fill specializations. Integration is one build setting in `project.yml`; XcodeGen 2.45.4 natively recognizes `.icon` bundles.

**Tech Stack:** SVG (hand-authored), Icon Composer `icon.json` (Xcode 26 format), XcodeGen 2.45.4, xcodebuild / simctl / assetutil for verification.

**Spec:** `docs/superpowers/specs/2026-07-10-app-icon-design.md`

## Global Constraints

- Deployment target iOS 26.0; icon must render in Default, Dark, Clear (light/dark), and Tinted (light/dark) modes.
- Glyph geometry in 160-unit reference space, scaled ×6.4 to a 1024 px canvas. All strokes: width 8 (160-space), round caps, single color.
- Speech line opacity 1.0; transcript and closing lines opacity 0.8.
- Palette — Default background gradient `#FDFDFC → #F2F2EF`, glyph `#232752`; Dark background gradient `#1A1D3C → #0B0C20`, glyph `#F2F2EF`.
- Deliverable is `Sotto/AppIcon.icon` (directory: `icon.json` + `Assets/glyph.svg`). **No asset catalog.**
- The Xcode project is generated: never edit `Sotto.xcodeproj` directly (it is gitignored); edit `project.yml` and run `xcodegen generate`.
- Commit messages: plain, no Claude attribution or Co-Authored-By trailers.
- Format ground truth (extracted from Icon Composer 26.6's `IconComposerFoundation` framework): keys `fill`, `linear-gradient`, `extended-srgb`, `fill-specializations`, `appearance`, `value`, `solid`, `groups`, `layers`, `image-name`, `name`, `supported-platforms`, `squares`; appearance values `dark`, `tinted`; layer images live in an `Assets/` subdirectory inside the bundle.
- **Fallback (from spec):** if the hand-written `icon.json` fails to parse in Icon Composer or at build time, assemble the same two layers manually in Icon Composer (`/Applications/Xcode.app/Contents/Applications/Icon Composer.app`) and save over `Sotto/AppIcon.icon`. The SVG artwork and colors are unchanged either way.

### Task 1: Author the `AppIcon.icon` bundle

**Files:**
- Create: `Sotto/AppIcon.icon/Assets/glyph.svg`
- Create: `Sotto/AppIcon.icon/icon.json`

**Interfaces:**
- Consumes: geometry and palette from the spec (copied verbatim below — no need to re-derive).
- Produces: `Sotto/AppIcon.icon` bundle that Task 2 references by path and Task 3 compiles. The icon's asset name is `AppIcon` (the bundle's basename), which Task 2's build setting must match exactly.

- [ ] **Step 1: Write the glyph SVG**

Create `Sotto/AppIcon.icon/Assets/glyph.svg` (the Write tool creates intermediate directories automatically):

```svg
<svg xmlns="http://www.w3.org/2000/svg" width="1024" height="1024" viewBox="0 0 1024 1024">
  <g transform="scale(6.4)" fill="none" stroke="#000000" stroke-width="8" stroke-linecap="round">
    <path d="M26 52 Q33 30 40 52 Q47 74 54 52 Q60 36 66 52 Q72 64 78 52 L134 52"/>
    <path d="M26 86 L134 86" stroke-opacity="0.8"/>
    <path d="M26 120 L98 120" stroke-opacity="0.8"/>
  </g>
</svg>
```

Notes for the implementer:
- The three paths are the spec's speech line, transcript line, closing line, verbatim in 160-unit space; the `scale(6.4)` group maps them onto the 1024 canvas. Do not pre-multiply the coordinates.
- The stroke color is a placeholder — Icon Composer replaces layer color via `fill` in `icon.json`; the SVG only contributes shape and per-element alpha (`stroke-opacity`), which is what carries the 0.8 hierarchy into Tinted mode.

- [ ] **Step 2: Verify the SVG is well-formed XML**

Run: `xmllint --noout Sotto/AppIcon.icon/Assets/glyph.svg && echo OK`
Expected: `OK` (no output from xmllint means no XML errors)

- [ ] **Step 3: Write `icon.json`**

Create `Sotto/AppIcon.icon/icon.json`. Color strings are `extended-srgb:R,G,B,A` with components in 0–1 (conversions from the spec's hex values are already done here — verify against Global Constraints, don't recompute):

```json
{
  "fill" : {
    "linear-gradient" : [
      "extended-srgb:0.99216,0.99216,0.98824,1.00000",
      "extended-srgb:0.94902,0.94902,0.93725,1.00000"
    ]
  },
  "fill-specializations" : [
    {
      "appearance" : "dark",
      "value" : {
        "linear-gradient" : [
          "extended-srgb:0.10196,0.11373,0.23529,1.00000",
          "extended-srgb:0.04314,0.04706,0.12549,1.00000"
        ]
      }
    }
  ],
  "groups" : [
    {
      "layers" : [
        {
          "image-name" : "glyph.svg",
          "name" : "glyph",
          "fill" : {
            "solid" : "extended-srgb:0.13725,0.15294,0.32157,1.00000"
          },
          "fill-specializations" : [
            {
              "appearance" : "dark",
              "value" : {
                "solid" : "extended-srgb:0.94902,0.94902,0.93725,1.00000"
              }
            }
          ]
        }
      ]
    }
  ],
  "supported-platforms" : {
    "squares" : "shared"
  }
}
```

Color key: top-level `fill` = `#FDFDFC → #F2F2EF` (Porcelain, light); its dark specialization = `#1A1D3C → #0B0C20`; glyph `solid` = `#232752` indigo ink; glyph dark specialization = `#F2F2EF`. Clear and Tinted modes need no entries — the system derives them from the mono-color glyph.

- [ ] **Step 4: Verify the JSON parses**

Run: `python3 -m json.tool Sotto/AppIcon.icon/icon.json > /dev/null && echo OK`
Expected: `OK`

- [ ] **Step 5: Open in Icon Composer — parse & preview gate**

Run: `open -a "/Applications/Xcode.app/Contents/Applications/Icon Composer.app" Sotto/AppIcon.icon`

Expected: Icon Composer opens the document without an error dialog and previews the mark — wavy first line settling into a rule, two quieter lines below, porcelain background. Use the appearance switcher to eyeball Default / Dark / Clear / Tinted.

**This is a human checkpoint — ask Connor to confirm the preview looks right before committing.** If Icon Composer refuses to open the document, apply the fallback from Global Constraints: rebuild the document in Icon Composer by hand from `Assets/glyph.svg` (background fill + one image layer, colors from the Color key above), save, and diff the app-generated `icon.json` against ours to learn the corrected schema.

If Icon Composer normalizes or rewrites `icon.json` on save, keep the app-written version — both workflows edit the same file by design.

- [ ] **Step 6: Commit**

```bash
git add Sotto/AppIcon.icon
git commit -m "feat: add AppIcon.icon — wave-becomes-text mark, porcelain palette"
```

### Task 2: Wire the icon into the XcodeGen build

**Files:**
- Modify: `project.yml` (Sotto target, lines 19–28 — sources block and new settings block)

**Interfaces:**
- Consumes: `Sotto/AppIcon.icon` from Task 1 (basename `AppIcon` is the asset name).
- Produces: a generated `Sotto.xcodeproj` whose Sotto target has `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon` and includes `AppIcon.icon` in its Resources build phase. Task 3 builds this project.

- [ ] **Step 1: Baseline — confirm the icon is absent from the current project**

Run: `grep -c "AppIcon" Sotto.xcodeproj/project.pbxproj || echo ABSENT`
Expected: `0` and/or `ABSENT` (grep exits nonzero on no match). If `Sotto.xcodeproj` doesn't exist locally, run `xcodegen generate` first.

- [ ] **Step 2: Add the app-icon build setting to the Sotto target**

In `project.yml`, the Sotto target (`targets: → Sotto:`) currently has no `settings:` block (the project-level one at the top sets Swift options). Add one directly under `platform: iOS`:

```yaml
  Sotto:
    type: application
    platform: iOS
    settings:
      base:
        ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon
    sources:
      - path: Sotto
        excludes:
          - "Resources"
```

Do not touch the `sources:` block yet — XcodeGen 2.45.4 has built-in Icon Composer support (its changelog: "Added built in `.icon` folder support for IconComposer #1600"), so the existing `- path: Sotto` glob should pick the bundle up correctly.

- [ ] **Step 3: Regenerate the project**

Run: `xcodegen generate`
Expected: `Generated project at .../Sotto.xcodeproj` with no warnings about AppIcon.icon.

- [ ] **Step 4: Verify the generated project**

Run: `grep -E "AppIcon" Sotto.xcodeproj/project.pbxproj`
Expected: an `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon;` line per build configuration, a file reference for `AppIcon.icon`, and a PBXBuildFile entry placing `AppIcon.icon` in the Resources build phase (grep for context with `-B2 -A2` if membership is unclear).

**Fallback if the glob mishandled the bundle** (e.g., `icon.json`/`glyph.svg` appear as individual files in the pbxproj): exclude it from the glob and add it explicitly, mirroring the CoreML model folder:

```yaml
    sources:
      - path: Sotto
        excludes:
          - "Resources"
          - "AppIcon.icon"
      - path: Sotto/AppIcon.icon
        buildPhase: resources
```

Then rerun Steps 3–4.

- [ ] **Step 5: Commit** (only `project.yml` — the xcodeproj is gitignored)

```bash
git add project.yml
git commit -m "build: compile AppIcon.icon as the app icon via xcodegen"
```

### Task 3: Build and verify the icon end-to-end

**Files:**
- None created or modified (verification only; screenshots go under the gitignored `DerivedData/`).

**Interfaces:**
- Consumes: the regenerated `Sotto.xcodeproj` from Task 2.
- Produces: verified evidence — compiled `Assets.car` containing the icon, and simulator home-screen screenshots in light and dark appearance.

- [ ] **Step 1: Build for simulator**

Run:
```bash
xcodebuild -project Sotto.xcodeproj -scheme Sotto \
  -destination 'platform=iOS Simulator,name=iPhone 17' \
  -derivedDataPath DerivedData build
```
Expected: `** BUILD SUCCEEDED **`. An actool failure mentioning AppIcon.icon means the icon.json schema was rejected — apply the Icon Composer fallback from Global Constraints, then rebuild.

- [ ] **Step 2: Confirm the icon was compiled into the asset catalog**

Run: `xcrun assetutil --info DerivedData/Build/Products/Debug-iphonesimulator/Sotto.app/Assets.car | grep -iE '"Name"|Icon' | sort -u`
Expected: entries naming `AppIcon` (Xcode 26 emits Icon Image assets from the .icon bundle). If `Assets.car` is missing entirely, the Resources phase didn't include the bundle — revisit Task 2 Step 4's fallback.

- [ ] **Step 3: Install on a simulator and screenshot the home screen (light)**

```bash
xcrun simctl boot "iPhone 17" 2>/dev/null; open -a Simulator
xcrun simctl install "iPhone 17" DerivedData/Build/Products/Debug-iphonesimulator/Sotto.app
sleep 3
xcrun simctl io "iPhone 17" screenshot DerivedData/icon-light.png
```

Then view `DerivedData/icon-light.png` (agents: use the Read tool — it renders images). Expected: the Sotto icon on the home screen — porcelain tile, indigo wave-into-text strokes, not a blank grid. The icon lands on the last home-screen page; if it isn't visible, uninstall/reinstall or check other pages in the Simulator window manually.

- [ ] **Step 4: Screenshot dark appearance**

```bash
xcrun simctl ui "iPhone 17" appearance dark
sleep 2
xcrun simctl io "iPhone 17" screenshot DerivedData/icon-dark.png
xcrun simctl ui "iPhone 17" appearance light
```

View `DerivedData/icon-dark.png`. Expected: the dark variant — deep indigo gradient tile with porcelain strokes (iOS icon appearance defaults to Automatic, which follows system appearance).

- [ ] **Step 5: Manual verification checklist — hand to Connor**

Simctl cannot drive Clear/Tinted icon modes or a physical device. Ask Connor to:
1. On simulator or device: long-press wallpaper → Customize → check all four icon appearance settings (Default, Dark, Clear, Tinted).
2. Check Settings app list and Spotlight for small-size legibility of the three-line mark.
3. On device (free-provisioning profile): build & run from Xcode, confirm the icon on the real home screen.

Report the results; no commit in this task.
